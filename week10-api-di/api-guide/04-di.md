# 04. DI（手動 → Hilt）

## 4-1. DI の目的（Flutter と同じ）

ViewModel が Repository の**具象クラス**を直接 new すると、テストで差し替えられない。
**インターフェースに依存し、実装は外から注入する**のが DI。

Flutter/Riverpod では Provider で定義 + `ref.read` で取得。
Android では **Hilt**（Dagger ベース）が標準。

## 4-2. 手動DI（小規模ならこれで十分）

```kotlin
// AppModule.kt — Riverpod の Provider 定義に相当
object AppModule {
    val apiService: GitHubApiService by lazy {
        RetrofitClient.apiService
    }
    val repository: GitHubRepository by lazy {
        GitHubRepositoryImpl(apiService)
    }
}

// ViewModel で使う
class SearchViewModel(
    private val repository: GitHubRepository = AppModule.repository,
) : ViewModel() { /* ... */ }
```

Riverpod で書くと：

```dart
final apiServiceProvider = Provider((ref) => RetrofitClient.apiService);
final repositoryProvider = Provider((ref) => GitHubRepositoryImpl(ref.read(apiServiceProvider)));
```

手動DI は `get_it` のサービスロケータと似た感覚。

## 4-3. Hilt（推奨）

Hilt は Dagger の上に乗った Android 専用 DI フレームワーク。
アノテーションを付けるだけで依存関係グラフが自動生成される。

### セットアップ

```kotlin
// build.gradle.kts (project)
plugins {
    id("com.google.dagger.hilt.android") version "2.52" apply false
}

// build.gradle.kts (app)
plugins {
    id("com.google.dagger.hilt.android")
    id("com.google.devtools.ksp")
}
dependencies {
    implementation("com.google.dagger:hilt-android:2.52")
    ksp("com.google.dagger:hilt-android-compiler:2.52")
    implementation("androidx.hilt:hilt-navigation-compose:1.2.0")
}
```

### 1. Application に @HiltAndroidApp

```kotlin
@HiltAndroidApp
class MyApplication : Application()
```

### 2. Module でインスタンスの作り方を定義

```kotlin
@Module
@InstallIn(SingletonComponent::class)   // アプリ全体でシングルトン
object NetworkModule {
    @Provides
    @Singleton
    fun provideApiService(): GitHubApiService {
        return RetrofitClient.apiService
    }

    @Provides
    @Singleton
    fun provideRepository(apiService: GitHubApiService): GitHubRepository {
        return GitHubRepositoryImpl(apiService)
    }
}
```

Riverpod の Provider 定義に相当。`@Provides` = 「この関数でインスタンスを作れ」。

### 3. ViewModel に @HiltViewModel + @Inject

```kotlin
@HiltViewModel
class SearchViewModel @Inject constructor(
    private val repository: GitHubRepository,   // Hilt が自動で注入
) : ViewModel() {
    // ...
}
```

### 4. Composable で hiltViewModel()

```kotlin
@Composable
fun SearchScreen(
    viewModel: SearchViewModel = hiltViewModel(),   // Hilt が自動でインスタンスを提供
) {
    val uiState by viewModel.uiState.collectAsStateWithLifecycle()
    // ...
}
```

## 4-4. Riverpod ↔ Hilt 対応表

| Riverpod | Hilt | 説明 |
|---|---|---|
| `Provider((ref) => ...)` | `@Provides fun ...()` | インスタンス定義 |
| `ref.read(provider)` | `@Inject constructor(dep)` | 依存を取得 |
| `ProviderScope` | `@InstallIn(Scope)` | スコープ |
| `autoDispose` | `@ActivityRetainedScoped` 等 | ライフサイクル |
| `overrideWithValue` | `@TestInstallIn` / テスト Module | テスト用差し替え |
| `family` | `@AssistedInject` | 引数付き |

## 4-5. テストでの差し替え

```kotlin
@Module
@TestInstallIn(
    components = [SingletonComponent::class],
    replaces = [NetworkModule::class],
)
object FakeNetworkModule {
    @Provides
    @Singleton
    fun provideRepository(): GitHubRepository = FakeGitHubRepository()
}
```

Riverpod の `overrideWithValue` と同じ目的。テスト時だけ Module を差し替える。
